С выходом SeaTable 6.2 [конструктор приложений]({{< relref "help/app-builder" >}}) получает новый тип страниц: **страницы HTML** позволяют создавать полностью индивидуальные страницы на основе HTML, JavaScript и CSS. Благодаря этому в области визуализации данных исчезают практически все ограничения. Веб-формы также можно оформить в точном соответствии с Вашими требованиями, и даже сложные взаимодействия можно без труда реализовать. «Невозможного» больше не существует!

Впрочем, и в остальном конструктор приложений находится в центре внимания версии 6.2. Тип страниц [Отдельная запись]({{< relref "help/app-builder/seitentypen-in-universellen-apps/seiten-vom-typ-einzelner-datensatz-in-universellen-apps" >}}) получает сразу тройное улучшение: теперь все значения столбцов можно единообразно отображать в виде текста, универсально форматировать и — часто высказываемое пожелание — редактировать прямо на странице «в режиме inline».

Другие [типы страниц]({{< relref "help/app-builder/seitentypen-in-universellen-apps" >}}) выигрывают от расширенного управления правами доступа для столбцов связей. Кроме того, небольшие изменения дизайна обеспечивают большую наглядность: приложения без страниц помечаются как таковые, чтобы избежать недоразумений, а аватар вместе с уведомлениями приложения нашёл новое место.

[Столбцы с формулами]({{< relref "help/base-editor/spaltentypen/die-formel-spalte" >}}) получают с SeaTable 6.2 сразу две новые возможности: новая функция **datetimeFormat()** преобразует значения даты в строки со свободно определяемым форматом. Если предпочитаемый Вами формат даты до сих пор не предлагался в SeaTable, теперь это в прошлом. Кроме того, строковые формулы теперь поддерживают **переносы строк**, благодаря чему тексты из разных столбцов можно компоновать значительно гибче.

Наконец, SeaTable 6.2 улучшает совместную работу: [внешние ссылки]({{< relref "help/startseite/freigaben/externer-link-erklaert" >}}) и [ссылки-приглашения]({{< relref "help/startseite/freigaben/einladungs-link-erklaert" >}}) теперь можно снабдить описанием, чтобы цель и предполагаемых получателей можно было указать прямо у ссылки. Кроме того, была переработана и сделана более наглядной лента для баз и представлений, открываемых по внешним ссылкам.

[Журнал изменений (changelog)](https://cloud.seatable.io/dtable/view-external-links/c9124bcd934b47bc8f30/) — как всегда — фиксирует все важные изменения.

Для [SeaTable Server]({{< relref "pages/product/seatable-server" >}}) версия 6.2 доступна для загрузки с сегодняшнего дня в [Docker-репозитории SeaTable](https://hub.docker.com/r/seatable/seatable-enterprise).

Обновление [SeaTable Cloud]({{< relref "pages/product/seatable-cloud" >}}) состоится **4 августа**. С этим обновлением месячная квота на [выполнение Python-скриптов]({{< relref "help/skripte/allgemein/skript-manuell-per-schaltflaeche-oder-automation-ausfuehren" >}}) становится динамической. По аналогии с вызовами API или автоматизациями общая квота для Python-скриптов привязывается к размеру команды. Команда Plus получает 250 запусков Python-скриптов, команда Enterprise — 1000 запусков Python на пользователя в месяц. Для подписки Free по-прежнему действует фиксированное ограничение в 100 запусков в месяц.

{{< warning headline="Breaking Changes в Python-скриптах" >}}

Возвращаемые значения объекта контекста и метода запроса для типов столбцов Дата, Формула, а также Формула для связей в версии 6.2 приводятся к единому виду. Это может привести к тому, что выполнение Ваших скриптов завершится с ошибками. Пожалуйста, проверьте и обновите Ваши Python-скрипты.

{{< /warning >}}

## Новый тип страниц приложения: страница HTML

С помощью **конструктора приложений SeaTable** можно создавать мощные приложения — без какого-либо программирования. Благодаря предопределённым типам страниц за короткое время создаётся полнофункциональное веб-приложение. Однако у такой эффективности есть своя цена: хотя **10 типов страниц** предлагают множество возможностей настройки, они задают структуру и поведение страницы. Индивидуальные настройки, выходящие за эти рамки, до сих пор были невозможны.

Хотите использовать тип диаграммы, который SeaTable не поддерживает? Создать веб-форму с высокой степенью индивидуализации? Встроить интерактивную схему с кликабельными элементами? Или объединить несколько представлений — например, таблицу и отдельную запись — на одной странице? С новым типом страниц **страница HTML** эти и многие другие требования можно реализовать.

![Новый тип страниц приложения: страница HTML](HTMLPage.png)

Страницы HTML могут отображать **статическое содержимое**, однако свой потенциал они раскрывают в сочетании с **данными базы**. Как и другие типы страниц конструктора приложений, они могут получать данные из базы и изменять записи в базе. Зато в **оформлении пользовательского интерфейса** Вы практически свободны. То, что можно реализовать с помощью HTML, CSS и JavaScript, в принципе можно воплотить и в виде страницы HTML в конструкторе приложений.

Пусть Вас не смущает название. Страницы HTML поддерживают не только **HTML**, но и **JavaScript, и CSS**. Весь код страницы загружается в приложение в виде пакета (bundle). Принцип работы, возможные подходы к разработке, а также справочник соответствующего Software Development Kit (SDK) Вы найдёте в [Руководстве разработчика SeaTable](https://developer.seatable.com/html-pages/ 'SeaTable Developer Manual'). Там же Вы найдёте и пример страницы.

В настоящее время этот тип страниц ориентирован в первую очередь на **пользователей с опытом программирования**. Уже разрабатывается функция ИИ, с помощью которой в будущем страницы HTML можно будет создавать и на естественном языке, без знаний программирования.

## Улучшенный тип страниц приложения: Отдельная запись

Тип страниц **Отдельная запись** получает в SeaTable 6.2 сразу несколько улучшений. До сих пор редактировать записи можно было только через детали строки, которые сначала нужно было открыть кнопкой «Редактировать запись». Это означало дополнительные клики и ненужный разрыв в удобстве работы в процессе редактирования.

С SeaTable 6.2 этот промежуточный шаг остаётся в прошлом. Если у пользователя есть необходимые права на редактирование, поля теперь можно редактировать прямо на странице. **Клик по полю активирует режим редактирования**, введённые данные проверяются, как обычно, а изменения сохраняются немедленно. Это обеспечивает заметно более плавный и интуитивно понятный рабочий процесс.

![Редактирование inline на страницах отдельной записи](SingleRecordPage_InlineEditing.png)

Расширено и отображение. **Типы столбцов** с собственным визуальным представлением — например, метки столбцов [Одиночный выбор]({{< relref "help/base-editor/spaltentypen/die-einfachauswahl-spalte" >}}) или [Сотрудник]({{< relref "help/base-editor/spaltentypen/die-mitarbeiter-spalte" >}}) — теперь можно при желании отображать в виде обычного текста. Кроме того, для всех значений, отображаемых в виде текста, доступен единый набор **параметров форматирования**: размер шрифта, цвет шрифта, выравнивание и фон можно единообразно настраивать независимо от исходного типа столбца. Так Вы сможете оформлять страницы с единообразным и аккуратным внешним видом.

## Расширенные права доступа для связей в приложении

SeaTable 6.2 закрывает пробел в управлении правами доступа [столбцов связей]({{< relref "help/base-editor/spaltentypen/die-verknuepfungsspalte" >}}) в конструкторе приложений. С помощью нового права **Удалять существующие записи** можно целенаправленно определить, разрешено ли пользователям удалять существующие связи.

До сих пор любой пользователь с правом редактирования мог удалять существующие связи. Теперь этим поведением можно управлять для столбца связей независимо от остальных [прав доступа к странице]({{< relref "help/app-builder/einstellungen/seitenberechtigungen-in-einer-universellen-app" >}}).

![Новое право в приложении: удаление существующих связей](AppBuilder_RemoveExistingRecords.png)

Теперь Вы можете всесторонне управлять правами доступа для столбцов связей: в зависимости от конфигурации пользователи могут создавать новые записи в связанной таблице, связывать существующие записи, изменять и удалять их — каждое действие со своими собственными правами. Это обеспечивает **значительно более точное управление правами на редактирование в приложениях**.

## Новая функция в столбце с формулой

Новая функция **datetimeFormat()** помогает Вам отображать значения даты именно так, как Вам нужно. Первый параметр содержит форматируемую дату или ссылку на столбец, содержащий значение. Во втором параметре Вы задаёте целевой формат.

![Формула для форматированных значений даты](FormulaColumn_datetimeFormat.png)

В строковых формулах Вы теперь можете вставить **перенос строки** с помощью «\n» или '\n'. Это особенно полезно, когда Вы объединяете несколько столбцов, но не хотите записывать значения в одну строку. Наглядный пример — адрес:

## Улучшения во внешних ссылках и ссылках-приглашениях

Наконец, SeaTable 6.2 припас оптимизацию для **внешних ссылок и ссылок-приглашений**. Теперь Вы можете снабдить их **описанием**. Так, например, цель и предполагаемых получателей можно указать прямо у ссылки. Кроме того, мы переработали и сделали более наглядной ленту для баз и представлений, открываемых по внешним ссылкам.

## И ещё кое-что

- **Улучшенный журнал базы**:​ [Журнал базы]({{< relref "help/base-editor/historie-und-versionen/aenderungen-ueber-die-logs-rueckgaengig-machen" >}}) был переработан в нескольких местах. Поля поиска облегчают выбор в больших базах, выпадающие меню имеют единообразный дизайн, а действия в удалённых таблицах теперь тоже можно фильтровать.

- **Улучшенный виджет даты​**: Виджет даты был оптимизирован с точки зрения управления и отображения и обеспечивает ещё более удобную работу со [значениями даты]({{< relref "help/base-editor/spaltentypen/die-datum-spalte" >}}).

- **Эмодзи в комментариях​**: [Комментарии]({{< relref "help/base-editor/zeilen/zeilen-kommentieren" >}}) теперь поддерживают эмодзи. Это делает отзывы, реакции и совместную работу ещё более выразительными.

## Изменение возвращаемых значений в Python-скриптах

С целью приведения форматов к единому виду в SeaTable 6.2 обновляются возвращаемые значения некоторых типов столбцов для объекта контекста и метода запроса. Это может потребовать доработки Ваших Python-скриптов.

Сопоставление возвращаемых значений в SeaTable 6.1 и SeaTable 6.2 Вы найдёте на [Форуме SeaTable](https://forum.seatable.com/t/important-changes-related-to-python-client-in-seatable-6-2/7435).

Для скриптов, выполняемых вручную, то есть скриптов, которые Вы запускаете с помощью кнопки или в Python-редакторе SeaTable, Вы должны учесть все указанные там изменения. Скрипты, выполняемые через автоматизацию, Вам нужно доработать только в части изменений, связанных с методом запроса.
