SeaTable wirkt auf den ersten Blick wie eine Tabellenkalkulation, ist aber eine **relationale Datenbank**. Das heißt: Die Tabellen in SeaTable stehen nicht isoliert für sich, sondern können mithilfe von **Verknüpfungsspalten** miteinander in Beziehung gesetzt werden. Dies funktioniert sowohl mit Datensätzen innerhalb einer Tabelle als auch über mehrere Tabellen hinweg.

Da man die Verknüpfung von Datensätzen in SeaTable mit speziellen Spalten umsetzt, finden Sie alle weiteren Erläuterungen rund um die [Verknüpfungsspalte]({{< relref "help/base-editor/spaltentypen/die-verknuepfungsspalte" >}}) im gleichnamigen Artikel. 

Zu den Besonderheiten von [Verknüpfungen innerhalb einer Tabelle]({{< relref "help/base-editor/tabellen/verknuepfungen-innerhalb-einer-tabelle" >}}) finden Sie eine separate Anleitung.

Für einen Einstieg in den technischen Hintergrund empfehlen wir Ihnen den [Artikel über relationale Datenbanken]({{< relref "posts/relationale-datenbank" >}}).
