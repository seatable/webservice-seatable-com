À primeira vista, o SeaTable parece uma folha de cálculo, mas é uma **base de dados relacional**. Isso significa: as tabelas no SeaTable não estão isoladas, mas podem ser relacionadas entre si com **colunas de ligação**. Isto funciona tanto com registos dentro de uma tabela como entre várias tabelas.

Uma vez que a ligação de registos no SeaTable é feita com colunas especiais, encontrará todas as explicações adicionais sobre a [coluna de ligação]({{< relref "help/base-editor/spaltentypen/die-verknuepfungsspalte" >}}) no artigo com o mesmo nome.

Para as particularidades das [ligações dentro de uma tabela]({{< relref "help/base-editor/tabellen/verknuepfungen-innerhalb-einer-tabelle" >}}) encontrará um guia separado.

Para uma introdução ao contexto técnico, recomendamos o [artigo sobre bases de dados relacionais]({{< relref "posts/relationale-datenbank" >}}).
