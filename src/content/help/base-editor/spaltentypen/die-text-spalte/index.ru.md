Текстовая колонка относится к самым базовым **типам колонок** в SeaTable и отображается на первом месте в меню выбора типа колонки.

![Обычная текстовая колонка](images/select-regular-text-column.png)

По умолчанию SeaTable автоматически добавляет в каждую вновь созданную таблицу **текстовую колонку** в качестве первой колонки. Кроме того, Вы можете вручную добавить в таблицу любое количество текстовых колонок. Как это сделать, Вы узнаете [здесь]({{< relref "help/base-editor/spalten/hinzufuegen-einer-spalte" >}}).

![Текстовая колонка](images/text-column-new.png)

**Первая колонка** таблицы имеет несколько [особенностей]({{< relref "help/base-editor/spalten/die-besonderheiten-der-ersten-spalte" >}}), о которых Вы можете прочитать в статье по ссылке.

## Использование текстовой колонки

Текстовая колонка особенно хорошо подходит для сбора и [сортировки]({{< relref "help/base-editor/ansichten/sortieren-von-eintraegen-in-einer-ansicht" >}}) коротких буквенно-цифровых строк, например, **терминов**, **имён**, **паролей**, **автомобильных номеров** или **IBAN**.

![Пример применения обычной текстовой колонки](images/regular-text-example.png)

{{< warning  headline="UTF-8"  text="SeaTable поддерживает все символы стандарта UTF-8, так как UTF-8 является наиболее распространённой в мире кодировкой символов для интернета." />}}

При использовании колонки имеет смысл делать записи **короткими**, так как без увеличения ширины колонки можно отобразить только **ограниченное количество символов**.

![Слишком длинные записи не могут быть полностью отображены в текстовой колонке без увеличения ширины колонки.](images/use-short-entries.png)

## Другие текстовые типы колонок

Обратите внимание, что для записей в текстовой колонке **опции форматирования недоступны**. Если Вы хотите вводить структурированные тексты с переносами строк, списками, изображениями и т. д., Вам следует использовать колонку [Форматированный текст]({{< relref "help/base-editor/spaltentypen/die-spalte-formatierter-text" >}}).

В SeaTable есть ещё три текстовых типа колонок для особых случаев использования:
- [колонка электронной почты]({{< relref "help/base-editor/spaltentypen/die-e-mail-spalte" >}})
- [колонка URL]({{< relref "help/base-editor/spaltentypen/die-url-spalte" >}})
- [колонка номера телефона]({{< relref "help/base-editor/spaltentypen/die-telefonnummer-spalte" >}})

{{< warning  headline="Примечание" >}}
Существует множество текстовых функций для [колонок-формул]({{< relref "help/base-editor/spaltentypen/die-formel-spalte" >}}), которые ведут себя как текстовые колонки, если результаты являются **строками**.
{{< /warning >}}

## Установка значения по умолчанию

Для каждой текстовой колонки можно задать предустановленное [значение по умолчанию]({{< relref "help/base-editor/spalten/standardwert-fuer-eine-spalte-festlegen" >}}). Оно автоматически вводится в каждую новую строку таблицы.

Если в качестве значения по умолчанию указать ссылку **{creator.name}** или **{creator.id}**, то автоматически вводится **имя** или **идентификатор пользователя**, добавившего строку.

![Ссылка на имя пользователя со значением по умолчанию](images/Set-creator-name-as-default-value.png)

## Проверка ввода

При использовании текстовых колонок в таблицах у Вас есть возможность проверки записей. С помощью валидации, которая поддерживает регулярные выражения, можно **проверить значения ячеек** и выделить ячейки с содержимым, отклоняющимся от допустимого формата.

Вы можете настроить проверку формата уже при создании текстовой колонки, активировав ползунок.

![Активация валидации ввода текстовых колонок](images/activate-validation-1.png)

Если Вы хотите проверить записи в уже созданной текстовой колонке, сначала нажмите на **символ треугольника** {{< seatable-icon icon="dtable-icon-drop-down" color="grey" >}} соответствующей колонки и выберите в выпадающем меню **Настроить тип колонки**.

1. Активируйте ползунок **Проверять ввод**
2. Определите **целевой формат**.
![Определение целевого формата](images/define-specific-format-2.png)
3. Подтвердите с помощью кнопки **Отправить**.

### Последствия валидации

После успешной проверки **ячейки** с **содержимым, отклоняющимся от** целевого формата, выделяются красным цветом.

![Ячейки, выделенные красным цветом, содержание которых отличается от целевого формата](images/marked-entries-which-not-match-the-format.png)

### Регулярные выражения

Для проверки Ваших записей в текстовых колонках SeaTable поддерживает **регулярные выражения**. В следующей таблице Вы найдёте несколько примеров:

| Регулярное выражение            | Функция                                                                       |
| ------------------------------- | ----------------------------------------------------------------------------- |
| \[123456\]                      | Проверьте, соответствует ли запись школьной оценке от 1 до 6.                  |
| \[1-9\]\[0-9\]?\[0-9\]?\[a-z\]? | Проверка формата немецкого номера дома (3 цифры + 1 буква)                     |
| \[0-9\]{5}                      | Проверка формата немецких почтовых индексов (5x число от 0 до 9)              |
| \[0-9/. \\-\]+                  | Проверка формата телефонного номера                                           |
| Max.\*Mustermann                | Поиск возможного отчества автора                                              |
