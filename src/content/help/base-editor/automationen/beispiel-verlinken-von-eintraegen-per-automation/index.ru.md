С помощью автоматизаций вам больше не придется связывать записи в таблицах вручную, эти действия могут выполняться автоматически. Просто определите соответствующую автоматизацию для любого табличного представления.

## Связывание записей с помощью автоматизации

![Блокировка и архивирование линий с помощью автоматизации](images/how-to-use-automations-for-locking-rows-3.png)

1. Нажмите на {{< seatable-icon icon="dtable-icon-rule" >}} в заголовке Base, а затем на **Automation Rules**.
2. Нажмите кнопку **Добавить правило**.
3. **Назовите** автоматизацию и определите **таблицу** и **представление, в** которых она должна работать.
4. Определите **триггерное событие, которое запускает** автоматизацию.
5. Нажмите **Добавить действие** и выберите **Добавить ссылки** в качестве автоматического действия.
6. Подтвердите с помощью кнопки **Отправить**.

## Создание автоматизации

Сначала определите **триггерное событие** для автоматизации.

![Триггерные события для блокировки и архивирования линий](images/trigger-options-for-archivating-rows.png)

{{< warning  headline="Важное замечание" >}}

Автоматическое действие **Добавить ссылки** доступно для вас **исключительно** доступна для следующих триггерных событий:

- "Записи удовлетворяют определенным условиям после обработки"
- "Добавлена новая запись"
- "Периодический триггер"

{{< /warning >}}

Затем в качестве автоматического действия выберите **"Добавить ссылки"**.

![Добавление автоматического действия](images/add-an-action.png) ![Добавление автоматического действия: "Добавить ссылки"](images/add-action-add-links.png)

Наконец, выберите **столбец**, который должен быть связан с выбранной таблицей, и определите **конкретное условие**, которое должно быть выполнено для запуска автоматизации.

![Определение автоматического действия](images/automated-actions-add-links.png)

В завершение подтвердите автоматизацию нажатием кнопки **Отправить**.

![Подтверждение автоматизации](images/confirm-the-automation.jpg)

## Пример использования

Конкретный **случай использования** для такого типа автоматизации может возникнуть, например, если вы работаете в бухгалтерии компании. Вы хотите вести как таблицу со всеми платежами, поступающими на счёт компании (**Payments**), так и таблицу со всеми счетами, которые должны быть оплачены в адрес компании (**Invoices**).

В таблице **Payments**, наряду с **номерами транзакций** платежей (P-0001 и т. д.), вы уже фиксируете **суммы** различных платежей (столбец **"Value"**).

![Пример таблицы Payments](images/Table-Payments-1.png)

Ваша таблица **Invoices**, в свою очередь, содержит **номера счетов** (столбец **"Invoice-No"**) и соответствующие **суммы счетов** (столбец **"Price Total"**).

![Пример таблицы "Invoices"](images/Table-Invoices.png)

С помощью **автоматизации** вы теперь хотите **связать** две таблицы **Payments** и **Invoices**, чтобы автоматически сопоставлять поступающие **платежи** со **счетами** на основании их номеров.

### Подготовка автоматизации

Прежде чем создавать автоматизацию, необходимо добавить ещё два столбца в таблицу **Payments**. В столбец **"Inv-No"** вы вручную вводите номера счетов, которые относятся к соответствующим платежам, — именно это впоследствии будет **запускать** автоматизацию. В столбце **"Invoices"** после создания автоматизации будут автоматически добавляться **связанные записи** из таблицы **Invoices**.

![Ручное расширение таблицы "Payments" двумя столбцами](images/Manuelle-Erweiterung-der-Tabelle-22Payments22.png)

В таблице **Invoices** вам также потребуется ещё один столбец (**"Collected Payments"**), в который в ходе автоматизации будут добавляться связанные записи из таблицы **Payments**.

![](images/Hinzufuegen-der-Spalte-22Collected-PAyments22.png)

После этого вы можете создать **автоматизацию** для таблицы **Payments**.

### Создание автоматизации

Сначала задайте имя автоматизации и выберите таблицу **Payments** вместе с одноимённым представлением, в котором должна действовать автоматизация.

![Определение автоматизации для примера использования](images/definition-of-the-automation-1.png)

В качестве **триггерного события** автоматизации выберите вариант **"Записи удовлетворяют определённым условиям после обработки"**. В качестве условия фильтра укажите, что оба столбца **"Inv-No"** и **"Value"** таблицы "Payments" **не пусты**. Это условие целесообразно, поскольку автоматизация должна запускаться, как только в этих столбцах появятся записи.

![Определение триггерного события для примера использования ](images/trigger-example-1-1.png) ![Определение триггерного события для примера использования ](images/trigger-example-2-1.png)

В качестве **автоматического действия** автоматизации в завершение определите действие **"Добавить связанный столбец в таблицу"**.

![Добавление автоматического действия: "Добавить ссылки"](images/add-action-add-links.png)

Столбец **"Invoices"** — это столбец, в который в ходе автоматизации будут добавляться связанные записи.

![Выбор столбца, в который добавляются связанные записи](images/automated-action-example-1.png)

На последнем шаге определения автоматизации следует задать ещё одно **условие**, которое должно выполняться для добавления связанной записи в столбец. В данном примере определяется, что каждая **запись**, добавляемая в столбец **"Inv-No"**, также должна присутствовать в столбце **"Invoice-No"** таблицы **Invoices**.

Конкретно это означает, что каждый **номер счёта**, который вы вручную присваиваете платежу в таблице **Payments**, уже должен присутствовать в таблице **Invoices**, чтобы соответствующие **записи** обеих таблиц могли быть **связаны** между собой.

![Определение заключительного условия запуска автоматизации](images/automated-action-example-2-1.png)

### Тестирование автоматизации

Если затем вы введёте в столбец **"Inv-No"** таблицы **Payments** номер счёта, который уже присутствует в таблице **Invoices**, соответствующая **связанная запись** будет автоматически добавлена в таблицу в столбце **"Invoices"**.

![Добавление связанных записей при запуске автоматизации](images/Ausloesen-der-Automation-Beispiel-1-1-1.png)

{{< warning  type="warning" headline="Важное замечание"  text="Если вы добавите в столбец **\\"Inv-No\\"** номера счетов, которых **пока нет** в таблице **Invoices**, автоматизация **не** будет запущена и соответственно **никакая** связанная запись в столбец добавлена не будет, поскольку соответствующей записи в таблице **Invoices** просто ещё не существует." />}}

При нажатии на связанную запись открывается окно, в котором вы можете просмотреть содержимое **связанной записи** из таблицы **Invoices**.

![При нажатии на связанную запись можно просмотреть содержимое связанного столбца в другой таблице.](images/Informationen-der-verlinkten-Spalte.png)

Кроме того, при запуске автоматизации **связанные записи** из таблицы **Payments** также добавляются и в таблицу **Invoices**. В столбце **"Collected Payments"** каждому номеру счёта автоматически сопоставляется соответствующий **номер платежа (P-000X)**. Благодаря **автоматизации** вы можете, таким образом, для каждого счёта видеть соответствующий **платёж** и быстро определять, какие счета оплачены, а какие ещё открыты.

![Добавление связанных записей при запуске автоматизации](images/Ausloesen-der-Automation-Beispiel-2.png)

Также в этой таблице нажатием на связанную запись можно открыть окно, в котором можно просмотреть содержимое **связанной записи** из таблицы **Payments**.

![При нажатии на связанную запись можно просмотреть содержимое связанного столбца в другой таблице.](images/Informationen-der-verlinkten-Spalte-2.png)
