С помощью автоматики вам больше не придется выполнять операции по обработке данных вручную, а можно сделать это автоматически. Разнообразие доступных [операций обработки данных]({{< relref "help/base-editor/datenverarbeitung/datenverarbeitungsoperationen-in-seatable" >}}) - от математических расчетов до создания связей между значениями в различных таблицах - открывает новые возможности для автоматизации рабочих процессов.

## Выполнить операцию обработки данных с помощью автоматизации

![Выполнять операции по обработке данных с помощью автоматизации](images/how-to-use-automations-for-locking-rows-3.png)

1. Нажмите на {{< seatable-icon icon="dtable-icon-rule" >}} в заголовке Base, а затем на **Automation Rules**.
2. Нажмите кнопку **Добавить правило**.
3. Дайте автоматике **имя** и определите **таблицу** и **представление, в** которых она должна срабатывать.
4. Определите **триггерное событие, которое запускает** автоматизацию.
5. Нажмите **Добавить действие** и выберите **Запустить операцию обработки данных** в качестве автоматического действия.
6. Выберите любую **операцию обработки данных**.
7. Выполните необходимые **настройки** (например, таблица, колонка источника, колонка результата) на **операции обработки данных**.
8. Подтвердите с помощью кнопки **Отправить**.

## Создание автоматизации

После того как вы определили, в какой **таблице** и **представлении** должна срабатывать автоматизация, определите [событие срабатывания]({{< relref "help/base-editor/automationen/automations-trigger" >}}). В настоящее время только **периодический триггер** может привести к операции обработки данных. Поэтому необходимо определить **время**, когда автоматизация должна срабатывать ежедневно, еженедельно или ежемесячно.

![Выбор триггера автоматизации для действия: Выполнить операцию обработки данных](images/select-trigger-for-data-operation-per-automation.png)

{{< warning  headline="Важное замечание"  text="Автоматизированное действие **Выполнение операции обработки данных** доступен для вас **исключительно** доступны при наступлении следующего триггерного события:" />}}

!["Периодический триггер]({{< relref "help/base-editor/automationen/automations-trigger" >}}#automations-trigger_periodischer_trigger)

Затем в качестве автоматического действия выберите **Выполнить операцию обработки данных** и выберите нужную [операцию обработки данных]({{< relref "help/base-editor/datenverarbeitung/datenverarbeitungsoperationen-in-seatable" >}}).

![Добавление автоматического действия](images/add-an-action.png)

![Выбор операции обработки данных, которая должна быть выполнена в качестве автоматического действия автоматизации](images/select-data-operation-for-automation.png)

На выбор вам предоставляются следующие **операции обработки данных**:

- [Расчёт накопленных значений]({{< relref "help/base-editor/datenverarbeitung/datenverarbeitung-kumulierte-werte-berechnen" >}})
- [Расчёт изменений]({{< relref "help/base-editor/datenverarbeitung/datenverarbeitung-veraenderungen-berechnen" >}})
- [Расчёт процентной доли]({{< relref "help/base-editor/datenverarbeitung/datenverarbeitung-prozentualen-anteil-berechnen" >}})
- [Расчёт рейтинга]({{< relref "help/base-editor/datenverarbeitung/datenverarbeitung-rangliste-berechnen" >}})
- [Перенос имён пользователей]({{< relref "help/base-editor/datenverarbeitung/datenverarbeitung-benutzernamen-uebertragen" >}})
- [Сравнить и скопировать]({{< relref "help/base-editor/datenverarbeitung/datenverarbeitung-vergleichen-und-kopieren" >}})

На следующем шаге задайте конкретные **настройки** для **операции обработки данных**, которая должна запускаться автоматизацией. В зависимости от выбранной операции вам будет предложено определить **таблицы** и/или **столбцы** (например, столбец-источник и столбец-результат), которые вы хотите использовать для математических вычислений или для установления отношений между значениями в разных таблицах.

![Конкретные настройки, которые необходимо выполнить для операции обработки данных](images/specific-settings-for-data-processing-operation.png)

В завершение подтвердите автоматизацию нажатием **Отправить**, чтобы сохранить её и запустить в заданное время триггера, или нажмите **Выполнить сейчас**, чтобы автоматизация была запущена **немедленно**.

![Сохранить автоматизацию или выполнить её немедленно](images/cancel-run-now-or-submit-automation.png)

## Пример использования

Конкретный случай использования для такого типа автоматизации может возникнуть, например, если вы хотите фиксировать в вашей таблице количество ежедневных посещений веб-сайта. Для этого вы хотите **суммировать** из строки в строку, в новом столбце, **числа посещений** за каждый отдельный день, которые фиксируются в существующем [числовом столбце]({{< relref "help/base-editor/spaltentypen/die-zahlen-spalte" >}}).

До сих пор [расчёт накопленных чисел посещений с помощью операции обработки данных]({{< relref "help/base-editor/datenverarbeitung/datenverarbeitung-kumulierte-werte-berechnen" >}}) был возможен только **вручную**. С помощью соответствующей автоматизации теперь стало возможно **автоматически** **каждый день** в фиксированное время рассчитывать накопленные числа посещений.

Для реализации вам сначала нужна таблица, в которой различные **дни** фиксируются в [столбце даты]({{< relref "help/base-editor/spaltentypen/die-datum-spalte" >}}), а **числа посещений** веб-сайта — в [числовом столбце]({{< relref "help/base-editor/spaltentypen/die-zahlen-spalte" >}}).

![Пример таблицы для случая использования автоматизации: Расчёт накопленных значений](images/Beispiel-Tabelle-automation-data-processing-kumulierte-werte-berechnen.png)

Затем добавьте в таблицу **второй числовой столбец**, в котором после каждого выполнения автоматизации будут рассчитываться накопленные значения. Новый, изначально пустой столбец _Cumulated Visitors_ выступает в операции обработки данных в роли **столбца-результата**, тогда как столбец _Unique Visitors_ является **столбцом-источником**.

![Добавление второго числового столбца](images/Beispiel-Tabelle-automation-data-processing-kumulierte-werte-berechnen-2.png)

Чтобы создать автоматизацию, выполните шаги, описанные в разделе [Создание автоматизации]({{< relref "help/base-editor/automationen/anlegen-einer-automation" >}}). Кроме того, выберите столбец _Unique Visitors_ в качестве **столбца-источника**, а вновь добавленный столбец _Cumulated Visitors_ — в качестве **столбца-результата**.

![Выбор столбца-источника и столбца-результата](images/auswahl-quell-und-ergebnisspalte-im-anwendungsfall.png)

После подтверждения автоматизации SeaTable ежедневно, в выбранное время, рассчитывает **накопленное число посетителей веб-сайта** и автоматически записывает его в столбец-результат. Автоматический расчёт накопленных значений, естественно, предполагает, что **число посетителей веб-сайта за предыдущий день** ежедневно, **до** выполнения автоматизации, записывается в столбец _Unique Visitors_.
