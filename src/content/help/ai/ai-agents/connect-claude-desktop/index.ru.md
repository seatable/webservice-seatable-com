В этом руководстве вы подключите Claude Desktop к вашей базе SeaTable. После настройки вы сможете задавать Claude вопросы о ваших данных и редактировать записи прямо из чата. Настройка занимает около пяти минут.

## Предварительные требования

- Аккаунт SeaTable Cloud с хотя бы одной базой
- Claude Desktop (доступен бесплатно на [claude.ai/download](https://claude.ai/download/))
- Аккаунт Anthropic (необходим для использования Claude)

{{< warning headline="Бесплатного тарифа достаточно, но требуется специальная настройка" >}}

Бесплатный аккаунт Anthropic в настоящее время поддерживает подключение к пользовательским MCP-серверам, однако их необходимо настраивать вручную через файлы конфигурации.

{{< /warning >}}

## Шаг 1: Создайте API-токен в SeaTable

API-токен предоставляет Claude доступ к определённой базе. Вы сами решаете, может ли Claude только читать или также записывать данные.
Инструкцию по созданию API-токена вы найдёте в статье [Создание API-токена]({{< relref "help/integrationen/seatable-api/erzeugen-eines-api-tokens" >}}).

{{< warning headline="Храните токен в безопасности" >}}

API-токен действует бессрочно и должен храниться как пароль. Если вы его потеряете или он случайно станет доступен, вы можете в любой момент удалить API-токен и создать новый.

{{< /warning >}}

## Шаг 2: Настройте Claude Desktop

Claude Desktop использует файл конфигурации JSON для подключения MCP-серверов.

1. Откройте Claude Desktop.
2. Перейдите в **Settings** → **Developer**

![Добавление пользовательского MCP-сервера в Claude Desktop](images/claude-add-custom-mcp.png)

3. Файл `claude_desktop_config.json` откроется в вашем текстовом редакторе.
4. Добавьте следующую конфигурацию:

```json
{
  "mcpServers": {
    "seatable": {
      "command": "C:\\Program Files\\nodejs\\node.exe",
      "args": [
        "C:\\Program Files\\nodejs\\node_modules\\npm\\bin\\npx-cli.js",
        "mcp-remote",
        "https://mcp.seatable.com/mcp",
        "--header",
        "Authorization: Bearer ВСТАВЬТЕ-ВАШ-API-ТОКЕН-СЮДА"
      ]
    }
  }
}
```

5. Замените `ВСТАВЬТЕ-ВАШ-API-ТОКЕН-СЮДА` на токен из шага 1.
6. Сохраните файл и перезапустите Claude Desktop.

Вы можете подключить несколько баз одновременно. Создайте отдельную запись в `mcpServers` для каждой базы с уникальным именем, например `seatable-crm` и `seatable-projects`.

## Шаг 3: Проверьте подключение

После перезапуска вы увидите подключение к **SeaTable** в разделе `Tools` в Claude Desktop. Это означает, что MCP-сервер подключён. Нажмите на него, чтобы увидеть доступные инструменты — вы должны найти инструменты SeaTable, такие как `list_tables`, `list_rows` или `query_sql`.

Также вы можете проверить статус подключения в настройках.

![Статус подключения к MCP-серверу](images/claude-mcp-connected.png)

Теперь задайте первый тестовый вопрос:

> *«Какие таблицы есть в моей базе?»*

Claude запросит структуру таблиц через MCP-сервер и перечислит все таблицы с их столбцами. Если это работает, подключение установлено.

## Задайте первые вопросы

Теперь вы можете задавать Claude вопросы о ваших данных, как если бы вы разговаривали с коллегой. Вот несколько примеров для начала:

- *«Сколько записей в таблице Контакты?»*
- *«Покажи мне все записи, где статус "Открыт".»*
- *«Подведи итоги по данным в таблице Выручка по месяцам.»*

Ваши вопросы должны относиться к таблицам и столбцам, которые действительно существуют в вашей базе. Если вы не уверены, сначала спросите Claude о структуре базы. Он знает ваши таблицы и столбцы и может подсказать, что доступно.

Вам не нужно вводить названия таблиц и столбцов точно. Claude распознаёт небольшие опечатки и исправляет их автоматически. Смело пишите «Контакты» вместо «контакты» или «Проекты» вместо «projects». Claude найдёт нужную таблицу.
